## Atualização 2026-07-12 (salvamento de chamada grande)
### O que foi alterado
O cliente deixou de espelhar `rowsJson` na query string ao salvar a chamada. O lote de alunos passa a ser enviado apenas no corpo do POST, evitando URL longa demais quando a turma tem muitos alunos.

### Conhecimento consolidado
Para `saveCall`, o payload grande deve ficar no corpo do POST e a URL deve carregar só o necessário para rota/compatibilidade. A duplicação de `rowsJson` na query string é uma armadilha recorrente para turmas grandes.

### Próximos passos
Se houver novo erro ao salvar chamada extensa, checar primeiro o tamanho da requisição no navegador e se o backend HTTP está aceitando o body sem redirecionar para GET.
